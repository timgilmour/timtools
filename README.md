# timtools

Skills and plugins for Claude Code that make development faster and more structured. These are opinionated tools built from real project experience — they encode workflows, architectural patterns, and project conventions so you can skip the boilerplate and get to the interesting work.

## Installation

Install via the Claude Code plugin system:

```bash
claude plugins install timtools@timgilmour
```

Or clone and reference locally:

```bash
git clone https://github.com/timgilmour/timtools.git
```

## What's Included

```text
timtools/
├── skills/
│   ├── ecs-architect/       # ECS game architecture suite (design, implement, review)
│   └── scaffold-project/    # Full-stack project scaffolding wizard
├── plugins/                 # Plugin manifests and configs
├── templates/               # Reusable templates
└── shared/                  # Shared resources and snippets
```

## Skills

### scaffold-project

**A project scaffolding wizard that sets up a fully configured Claude Code + VS Code + Obsidian workspace from an empty directory.**

Most Claude Code projects start the same way: initialize git, create a folder structure, configure permissions, set up documentation. `scaffold-project` automates all of it through an interactive questionnaire that asks about your project's name, language, delivery method, and goals, then generates everything in one pass.

The skill creates a complete Obsidian vault with 11 numbered directories (Inbox, Plans, Notes, Reference, Tasks, Skills, Code, Config, MCP, Output, Attachments), pre-configured with 21 community plugins for documentation workflows. It generates a `CLAUDE.md` with your project's conventions, an `AGENTS.md` with subagent routing guidance, and a `.claude/settings.local.json` with language-appropriate permissions. It even initializes persistent memory and runs an initial GitNexus index so code intelligence is available from commit one.

Language support covers TypeScript, Python, Go, and Rust out of the box, each with tailored app directory scaffolds and permission configs. The wizard also discovers your installed Claude Code plugins, lets you enable them per-project, and offers to browse the marketplace for plugins that match your project's goals. The result is a workspace where every tool — VS Code, Claude Code, Obsidian, GitNexus — is wired together and ready to use immediately.

**Usage:** `/scaffold-project`

### ecs-architect

**A three-skill suite for designing, implementing, and reviewing game systems using Entity-Component-System architecture. Targets TypeScript/Three.js and Godot/GDScript.**

ECS is a powerful architectural pattern for games and simulations, but it's easy to drift into anti-patterns — logic creeping into components, "manager" classes that mix state and behavior, implicit execution order through callbacks and event buses. The `ecs-architect` suite encodes a strict, opinionated interpretation of ECS drawn from Bevy, hecs, and Oxygengine, adapted for the TypeScript and Godot ecosystems. It gives Claude Code a shared vocabulary and a concrete set of rules to enforce, so the architecture stays clean as features grow.

The suite follows a design-first workflow with three sequential skills. **ecs-design** takes a feature description and produces a structured design document: components (pure data), systems (all logic), resources (scoped globals), channels (pull-based messaging), and a pipeline that reads like a sentence describing the data flow. **ecs-implement** takes that design and generates working scaffold code — a minimal World class with entity management, component storage, query engine, resource store, channel system, and pipeline wiring. For Three.js projects, it includes a RenderSync bridge pattern that treats the scene graph as a render output sink rather than the source of truth. **ecs-review** audits existing code against ECS principles using a severity-ranked checklist (critical/warning/suggestion), catching violations like logic in components, system-to-system references, implicit execution order, and callback-based messaging.

The architecture principles document at the heart of the suite defines seven core concepts (World, Entities, Components, Systems, Resources, Pipelines, Channels) with full code examples in both TypeScript and GDScript. It includes a decision guide for classifying new code ("Is this data or behavior? Is it shared? Does it need cross-system communication?"), a table of anti-patterns with fixes, and guidance on lazy query chains, archetypes, and world lifecycle management. The review skill can be used on any codebase, not just code generated by the suite — making it useful for auditing existing game projects against ECS best practices.

**Usage:**

```text
1. /ecs-design      → Produce a design doc for the feature
2. /ecs-implement   → Scaffold code from the design
3. /ecs-review      → Audit the result against ECS principles
```

**Supported targets:**

| Target     | Environment                       |
|------------|-----------------------------------|
| TypeScript | Web, Node, Vite, Astro, Three.js  |
| GDScript   | Godot 4.x                         |
